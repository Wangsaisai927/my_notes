## From Street to Simulator: Assessing Cyclist Movement Consistency Between Virtual and Real-World Environments
1. 文章情景：主要是在讨论一个问题：VR骑行到底和真实骑行像不像？换句话说就是：在“动作”和“体验”上，VR和现实差多少？人体运动非常复杂，比如骑车时身体怎么平衡、怎么转头看路、手脚怎么配合，这些在VR里可能会有偏差。现在很多研究都会用VR来做实验，比如研究交通行为、骑行安全之类的，因为VR安全、可控。但问题在于VR里的行为是不是真的能代表现实？如果这些差异很大，那用VR做研究可能就不太靠谱。所以文章是在验证VR和现实的匹配度。
2. 文章实验方法：对照实验 + 数据集。对照实验：一次在VR里骑车（用Unity做的模拟器） / 一次在真实世界骑车（同一条路线），而且尽量保证两边条件一致，比如路线长度、骑行姿势这些都对齐。在骑车过程中，他们在人体多个部位装了IMU传感器（头、身体、手、腿），用来记录运动数据（高频率、120Hz），同时还录了第一视角视频。骑完之后还让参与者填问卷，比如有没有晕、开不开心之类的。分析的时候，他们把IMU三轴数据处理成一个“运动强度”（？？？），然后分析：骑车节奏（比如踩踏是不是规律） / 左右腿是不是同步 / 上半身是不是稳定 / 头是怎么动的（看哪里）。用了一些统计方法（t检验、相关性分析）去比较VR和真实之间有没有显著差异。
3. 实验设计指标：运动相关的指标：踏频（衡量踩踏是不是有规律） / 方差（衡量动作稳不稳定） / 相关性（衡量左右腿是不是同步） / 角速度和加速度（衡量身体晃动、旋转情况）。视觉和行为相关指标：头部转动范围（左右看的幅度） / 转头频率（多久看一次周围）。主观指标：晕动症（SSQ） / 沉浸感 / 开心和挫败感。

## I Can’t Turn My Brain Off: Exploring Conversational AI for Self-Critical Thoughts in VR
1. 文章情景：人脑子里会有“自我批评”的想法（self-critical thoughts），很多人都会在晚上胡思乱想，比如自责、后悔、焦虑这些，而且这种时候往往没有人可以及时提供帮助（比如朋友不方便 / 心理咨询不现实）。虽然有一些方法，比如写日记、自我对话，但问题是：人很难在情绪很差的时候主动开始这些行为（感觉这个切入点很屌啊）。另外，如果没有引导，自我对话甚至可能会变成“越想越糟”。文章工作就是在解决能不能用VR+对话AI来帮人处理这些负面想法？
2. 文章工作：做了一个VR系统，对话AI引导用户说出负面想法，把这些负面想法变成一个可以操作的虚拟物体，用一些隐喻对这些想法进行操作操作，比如：把想法种下去后变成花 / 烧掉 / 想法像叶子一样飘走或顺水流走（这里有点牵强）。把抽象的心理问题变成可以操作的VR交互。
3. 实验（这个有点扯）：用VR（类似一个自然场景，比如森林、河流） / 用CAI（其实是Wizard-of-Oz，也就是人假装AI） / CAI会引导你说话，但不会太“有存在感”（避免压力）。实验是很小的pilot study。

## OHMNI: Hand-Tracked XR Circuit Prototyping with Integrated SPICE Simulation(这个文章有一个新的概念是XR)
1. 文章情景：主要是在讨论电路原型设计怎么更直观、2D仿真工具，虽然方便但不直观，尤其是在理解电路结构和电流流动方面比较抽象。虽然XR可以提供更沉浸的交互方式，但现有系统要么依赖PC，要么用手柄，不够自然，而且也缺少系统性的评估。文章在探索：能不能用纯手势+XR的方式来做电路搭建，同时还能实时仿真电路行为。
2. 文章工作：用Unity开发（XR Interaction Toolkit）做了一个XR系统，运行在Meta Quest 3上。用户可以直接用手来搭电路(比如用去抓取元件、连接引脚，操作开关之类的)，把搭面包板这件事搬到3D空间里做。同时系统集成了SPICE仿真，一旦电路连接发生变化，就会实时计算电压、电流等信息，并直接显示在电路上。
3. 实验：做了两个实验。一个是性能测试，包括帧率、GPU/CPU负载、长时间稳定性这些，主要是看这个系统在XR设备上能不能稳定运行。另一个是用户实验（20个学生），对比OHMNI和Tinkercad，指标包括：任务完成时间 / SUS可用性评分 / 用户主观反馈。结果是XR更慢（346s vs 216s）且可用性更低，但在空间理解上更有优势（有些人反而更快）。整体来看就是：交互更直观，但效率和易用性还不如传统2D工具。
